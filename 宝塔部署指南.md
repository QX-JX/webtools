# 宝塔面板 Docker 部署指南

## 准备工作

1. **确保宝塔面板已安装**：在服务器上安装并登录宝塔面板
2. **确保已安装 Docker 插件**：在宝塔面板的软件商店中搜索并安装 Docker 管理器
3. **上传项目文件**：将整个项目目录上传到服务器的指定位置，例如 `/www/wwwroot/webtools`

## 部署步骤

### 方法一：使用 Docker Compose 部署

1. **进入项目目录**：在宝塔面板中打开终端，进入项目目录
   ```bash
   cd /www/wwwroot/webtools
   ```

2. **构建并启动容器**：
   ```bash
   docker-compose up -d
   ```

3. **查看部署状态**：
   ```bash
   docker-compose ps
   ```

### 方法二：手动构建镜像并运行容器

1. **构建 Docker 镜像**：
   ```bash
   docker build -t kunqiong-webtools .
   ```

2. **运行容器**：
   ```bash
   docker run -d --name webtools -p 8080:80 --restart always kunqiong-webtools
   ```

## 访问项目

部署完成后，可以通过以下方式访问项目：

- **访问地址**：`http://服务器IP:8080`
- **默认端口**：8080（可在 docker-compose.yml 中修改）

## 项目结构说明

- **前端**：位于 `/app/frontend` 目录，使用 React 构建的静态文件
- **后端**：位于 `/app/backend` 目录，使用 Express 构建的 API 服务
- **Nginx**：用于代理前端和后端请求

## 配置说明

### 端口配置

- **容器内部**：Nginx 监听 80 端口
- **外部访问**：默认映射到主机的 8080 端口

### 环境变量

- `NODE_ENV=production`：设置为生产环境

## 常见问题排查

1. **容器启动失败**：
   - 查看容器日志：`docker logs webtools`
   - 检查端口是否被占用：`netstat -tlnp | grep 8080`

2. **API 无法访问**：
   - 检查后端服务是否正常运行：`docker exec -it webtools ps aux | grep node`
   - 检查 Nginx 配置是否正确：`docker exec -it webtools cat /etc/nginx/nginx.conf`

3. **前端页面无法加载**：
   - 检查前端文件是否正确复制：`docker exec -it webtools ls -la /app/frontend`
   - 检查 Nginx 访问日志：`docker exec -it webtools cat /var/log/nginx/access.log`

## 更新项目

1. **停止并删除容器**：
   ```bash
   docker-compose down
   ```

2. **更新项目文件**：将最新的代码上传到服务器

3. **重新构建并启动**：
   ```bash
   docker-compose up -d --build
   ```

## 注意事项

- 确保服务器防火墙已开放 8080 端口
- 建议使用域名访问，并在宝塔面板中配置反向代理
- 定期更新 Docker 镜像以获得最新的安全补丁
